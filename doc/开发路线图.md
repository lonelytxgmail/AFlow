## 2. 当前已实现能力基线

### 2.1 后端引擎

- DAG 工作流执行：`start / resume / retry / suspend / cancel / step`
- 条件分支路由（SpEL 表达式）
- **错误处理边（Try-Catch 语义）** — 节点失败可自动走 error 边到备选路径
- `forEach / while` 专用循环节点
- **子流程节点** — `subflow` 类型，支持嵌套 workflow 调用
- 节点执行路径记录
- 快照记录与节点 diff 查询
- 事件持久化
- 断点暂停 + 单步调试
- 挂起态上下文热更新
- 默认异步执行（Java 21 虚拟线程）
- SSE 实时事件推送（含 traceId 传递）
- **节点级执行超时** — `CompletableFuture.orTimeout()`
- **节点级自动重试** — 可配 `retryPolicy`，指数退避

### 2.2 节点能力

当前内置节点：

| 节点类型 | 说明 |
|---------|------|
| `http` | HTTP 请求 |
| `condition` | 条件分支 |
| `transform` | 数据转换 |
| `script` | 脚本执行（Groovy） |
| `assign` | 变量赋值 |
| `delay` | 延时等待 |
| `callback` | 外部回调挂起 |
| `log` | 日志输出 |
| `forEach` | 集合遍历循环 |
| `while` | 条件循环 |
| `composite` | 原子能力引用 |
| `agent` | AI Agent（ReAct 模式） |
| `subflow` | 子流程调用 ⭐ 新增 |

### 2.3 原子能力

- CRUD + 状态流转：`DRAFT / PUBLISHED / ARCHIVED`
- 已发布原子能力注册表
- 独立测试调用
- `composite` 节点引用执行 + 配置模板合并
- 输入 schema 必填校验
- 已发布原子能力自动暴露为 Agent Tool

### 2.4 Agent 能力

- ReAct 循环执行 + 工具调用与结果回填
- **动态 Tool 选择** — 支持静态列表 / SpEL 表达式 / 全量（`*`）
- **Tool 调用频率限制** — `maxCallsPerTool` / `maxTotalCalls`
- **Tool 执行超时** — 超时不终止循环，错误回传 LLM
- **Tool 结果截断** — 75% head + 15% tail + truncation marker
- **Token 预算管理** — 估算 token 数，自动裁剪早期消息
- **LLM 调用重试** — 装饰器模式，指数退避，区分瞬时/永久错误
- **输出 Guardrails** — JSON Schema 验证 + retry/fail/passthrough 策略
- 结构化事件：`AGENT_THINK / ACT / OBSERVE / DONE / TOOL_TIMEOUT / TOOL_RATE_LIMITED / TOKEN_PRUNE / OUTPUT_VALIDATION_FAILED`
- **LLM 调用指标** — AGENT_DONE 事件含 totalTokens、totalLlmCalls、totalLlmLatencyMs、modelVersion

### 2.5 可观测性

- **Micrometer Metrics** — 节点耗时分布、执行计数、活跃流程数、Agent 迭代数、Tool 调用数、LLM 调用延迟、Token 消耗
- **Prometheus 端点** — `/actuator/prometheus`
- **OpenTelemetry 追踪** — Micrometer Observation API + OTel bridge，支持 flow/node/llm/tool 四级 span
- **OTLP 导出** — 可配置推送到 Jaeger / Tempo
- **SSE 事件含 traceId** — 前端可关联追踪

### 2.6 DSL 模型

- `EdgeDefinition` 支持 `type` 字段：`normal`（默认）/ `error`
- `NodeDefinition` config 中支持 `timeout`、`retryPolicy`
- Agent 节点 config 支持：`toolTimeout`、`maxTokenBudget`、`toolResultMaxLength`、`toolRateLimit`、`outputSchema`、`outputValidation`

### 2.7 前端能力

- 流程定义列表 + 编辑页（DAG 拖拽 + JSON DSL + 节点配置面板）
- 实例列表 + 详情页（DAG 高亮 + 事件流 + 快照 diff + 调试操作 + Agent 轨迹）
- 原子能力管理页（CRUD + 发布/归档 + 独立测试）

> ⚠️ 前端尚未跟进后端新增能力（错误边可视化、子流程、Guardrails 配置等），后续需要前端重构设计。

## 3. 下一阶段路线图

### 阶段三：前端重构与体验收敛（最高优先级）

> 目标：前端需要重新设计以匹配后端当前能力水平

### 阶段五：定义与运行时契约收敛

- DSL 字段文档化 + 合同冻结
- 运行时恢复语义明确化
- API contract 稳定

### 阶段六：生产化治理（后排）

- 认证与权限
- 密钥管理
- 审计查询
- 幂等启动 + 恢复策略
- 多租户
- 定义版本治理
- **Human-in-the-Loop 审批节点**（设计已完成，实现后排）

### 阶段七：高级 Agent 能力（按需）

- Agent 记忆 / 跨执行上下文
- 多 Agent 协作
- Streaming Agent 输出
- 并行节点执行

## 5. 配置参考

```yaml
aflow:
  engine:
    node-timeout-ms: 300000          # 节点超时 5 分钟
    context-cache:
      max-size: 10000
      ttl-minutes: 30
  agent:
    tool-timeout-ms: 30000           # Tool 超时 30 秒
    token-budget: 100000             # Token 预算 ~100K
    tool-result-max-length: 4000     # 结果截断阈值
    llm:
      retry:
        max-attempts: 3
        initial-backoff-ms: 1000
        backoff-multiplier: 2.0

management:
  endpoints.web.exposure.include: health,info,prometheus,metrics
  tracing:
    sampling.probability: 1.0
  otlp.tracing.endpoint: http://localhost:4318/v1/traces
```

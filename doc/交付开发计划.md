# AFlow 交付开发计划

> 目标：从当前状态推进到**可交付状态**（可演示、可部署、前后端能力对齐）
> 排期单位：对话轮数（每轮 ≈ 一次完整的 spec→实现→验证 周期）
> 日期：2026-06-18

---

## 可交付状态定义

一个"可交付"的 Agentic Workflow 平台应满足：

1. ✅ 前端能力完整覆盖后端功能（结构化配置、错误边、子流程、Agent 高级配置）
2. ✅ 前端使用 TypeScript + 专用配置面板（非 JSON 编辑为主）
3. ✅ RAG 知识检索可用（Agent 可搜索知识库）
4. ✅ DAG 并行执行（无依赖节点并行跑）
5. ✅ 容器化一键启动（docker-compose）
6. ✅ 运行时监控面板可用
7. ✅ Human-in-the-Loop 审批节点可用
8. ✅ DSL 契约文档化 + API 稳定

---

## 总览：18 轮对话完成交付

```
Phase A: 前端重构基础 (轮 1-4)

```

## Phase C：RAG 知识检索（轮 10-12）

**目标**：创建 `aflow-knowledge` 模块，实现文档入库流水线

**任务**：

### 轮 11：知识库后端 — 检索 + Agent 集成

**目标**：实现检索流水线，将知识检索暴露为 Agent Tool

**任务**：
- [ ] 实现 `VectorSearchService`：向量相似度搜索（cosine, top_k）
- [ ] 实现 `HybridSearchService`：向量 + 关键词(PostgreSQL full-text) 混合检索
- [ ] 实现 `SearchReranker`：基于 cross-encoder 或 LLM 的重排序（可选）
- [ ] 创建 `KnowledgeRetrievalTool`：实现 `Tool` 接口，供 Agent 调用
  - name: `search_knowledge`
  - parameters: `{ query: string, knowledge_base_id?: string, top_k?: int }`
  - 返回：ranked chunks with metadata + relevance score
- [ ] 在 `ToolRegistry` 中自动注册所有知识库为可用 Tool
- [ ] 创建 `knowledge_retrieval` 节点类型（独立节点，非 Agent 内调用）
- [ ] 在 `aflow-components` 中注册新节点类型
- [ ] 配置属性：`aflow.knowledge.embedding-model`、`aflow.knowledge.default-top-k`

**产出**：Agent 可调用知识检索 Tool，也可作为独立节点使用

## Phase D：引擎增强（轮 13-15）

### 轮 13：DAG 并行执行

**目标**：无依赖节点自动并行执行

**任务**：
- [ ] 在 `DefaultWorkflowEngine` 中实现拓扑排序分波（wave decomposition）
- [ ] 同一 wave 内的节点通过 `CompletableFuture.allOf()` 并行执行
- [ ] 并行执行结果汇聚后才进入下一 wave
- [ ] 支持 `parallel` 节点类型（显式声明并行分叉）+ `join` 节点（汇合等待）
- [ ] 并行执行时每个节点独立超时、独立重试
- [ ] 并行分支部分失败时的策略：`failFast` / `waitAll` / `bestEffort`（可配）
- [ ] 发布 `PARALLEL_FORK` / `PARALLEL_JOIN` 事件
- [ ] 更新前端 DAG Viewer：并行节点水平排列，join 点有汇聚标识
- [ ] 属性测试：∀ DAG, 并行执行结果 = 串行执行结果（确定性保证）

**产出**：DAG 中无依赖节点自动并行，执行效率显著提升

## 依赖关系与关键路径

```
轮1 (TS迁移) ──→ 轮2 (布局) ──→ 轮3 (DAG增强) ──→ 轮5 (配置面板)
                                                        │
轮4 (API类型) ─────────────────────────────────────────┘
                                                        │
                                                        ▼
                                                   轮6 (变量) ──→ 轮7 (调试页)
                                                                       │
轮10 (RAG后端) ──→ 轮11 (RAG检索) ──→ 轮12 (RAG前端)                    │
                                                                       ▼
轮13 (并行执行) ─────────────────────────────────→ 轮18 (集成测试)
轮14 (审批节点) ─────────────────────────────────→ 轮18
轮15 (版本管理+LLM节点) ────────────────────────→ 轮18
                                                        │
轮8 (监控面板) ──→ 轮9 (测试打磨) ──→ 轮16 (容器化) ──→ 轮17 (文档) ──→ 轮18
```

**关键路径**：轮1→2→3→5→6→7→9→16→17→18

**可并行的工作流**：
- Phase C (RAG, 轮10-12) 可与 Phase B 后半段(轮7-9)并行
- 轮13(并行执行)、轮14(审批)、轮15(版本管理) 三者互相独立可并行
